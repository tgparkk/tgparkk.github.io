## 1. 기존 글 개선 (즉시 효과)

### 1-1. 대표이미지 추가 (우선순위 높음)

현재 53개 포스트 중 67%에 대표이미지가 없습니다. 대표이미지는:
- Google 검색결과에서 썸네일로 표시 → **CTR 2~3배 향상**
- SNS 공유 시 미리보기 카드에 표시
- og:image 태그로 자동 연동

**추가 방법:**
```yaml
---
layout: post
title: "포스트 제목"
image: /assets/images/posts/your-image.png  # 이 줄 추가
---
```

**우선 추가할 포스트 (트래픽 상위):**
1. 한국투자증권 Open API 파이썬 자동매매 시리즈 (전체)
2. 주식 단타 전략 소개
3. 한국투자증권 API로 자동매매 시스템 개발
4. BaseStrategy 가상장 시리즈

**이미지 권장 사항:**
- 크기: 1200x630px (OG 이미지 표준)
- 형식: PNG 또는 WebP
- 내용: 코드 스니펫 캡처, 다이어그램, 차트 등 기술적 이미지
- 도구: Canva (무료), Carbon (코드 이미지), Excalidraw (다이어그램)

### 1-2. excerpt 최적화

excerpt는 검색결과의 설명문(meta description)으로 사용됩니다.

**좋은 excerpt 작성법:**
- 120~160자 이내 (그 이상은 잘림)
- 핵심 키워드를 앞부분에 배치
- 독자가 클릭하고 싶게 만드는 문장

**나쁜 예:** "이번 포스트에서는 자동매매에 대해 알아보겠습니다."
**좋은 예:** "한국투자증권 Open API와 Python으로 실전 자동매매 시스템을 구축하는 방법. 환경설정부터 실시간 주문까지 단계별 가이드."

## 2. 신규 글 작성 체크리스트

### 2-1. 키워드 리서치 방법

글을 쓰기 전에 사람들이 실제로 검색하는 키워드를 확인하세요.

**무료 도구:**
1. **Google 자동완성** — 검색창에 키워드 입력 후 추천 검색어 확인
2. **Google Trends** (trends.google.co.kr) — 키워드 검색량 추이 비교
3. **네이버 키워드 도구** (searchad.naver.com) — 월간 검색량 확인 (네이버 광고 계정 필요, 무료)
4. **AnswerThePublic** (answerthepublic.com) — 키워드 관련 질문 목록

**기술 블로그에서 검색량 높은 패턴:**
- "OOO 사용법" / "OOO 예제" / "OOO 튜토리얼"
- "OOO vs OOO 비교"
- "OOO 에러 해결" / "OOO 오류"
- "OOO 설치" / "OOO 환경설정"

### 2-2. SEO 효과 높은 포스트 구조

```
# H1: 핵심 키워드를 포함한 제목 (1개만)

도입부 (2~3문장): 이 글에서 다루는 내용과 독자가 얻을 것

## H2: 주요 섹션 제목 (키워드 포함)

본문 내용...

### H3: 하위 섹션

본문 내용...

## H2: 다음 섹션

...

## 마무리 / 정리

핵심 내용 요약 + 다음 글 예고 (내부 링크)
```

**포스트 작성 전 체크리스트:**
- [ ] 제목에 핵심 키워드 포함 (한국어 검색 타겟)
- [ ] excerpt 120~160자 작성
- [ ] 대표이미지 준비
- [ ] tags에 관련 키워드 3~5개
- [ ] H2/H3 헤딩 구조 잡기
- [ ] 본문 최소 1,500자 이상 (기술 글은 3,000자+)

**포스트 작성 후 체크리스트:**
- [ ] 관련 기존 포스트로 내부 링크 1~2개 추가
- [ ] 코드 블록에 언어 지정 (```python, ```cpp 등)
- [ ] 이미지에 alt 텍스트 작성

### 2-3. 콘텐츠 아이디어 (검색량 기반)

현재 블로그 강점(주식 자동매매 + C++)을 살린 키워드 후보:

**주식/자동매매 (고트래픽):**
- "파이썬 주식 자동매매 만들기"
- "한국투자증권 API 사용법"
- "주식 백테스팅 파이썬"
- "퀀트 투자 전략 구현"
- "실시간 주식 데이터 수집"

**C++/시스템 (중트래픽, 꾸준한 검색):**
- "C++ 메모리 풀 구현"
- "C++ 멀티스레드 프로그래밍"
- "비동기 서버 C++ 구현"
- "게임 서버 아키텍처"

**개발 일반 (진입 키워드):**
- "GitHub Pages 블로그 만들기"
- "Jekyll 블로그 커스터마이징"
- "개발자 블로그 시작하기"

## 3. 내부 링크 전략

내부 링크는 SEO에서 매우 중요합니다. 구글 봇이 사이트 구조를 이해하고, 관련 페이지의 권위도를 높여줍니다.

### 3-1. 기본 원칙
- 모든 새 글에서 관련 기존 글로 **최소 1~2개** 링크
- 기존 인기 글에서 새 글로 **역링크** 추가
- 앵커 텍스트에 키워드 포함 (예: "자동매매 시스템 구축 방법" > "여기를 클릭하세요")

### 3-2. 시리즈 기능 활용
이미 series-banner 기능이 있으므로 적극 활용하세요.

```yaml
---
series: "한국투자증권 자동매매"
series_order: 3
---
```

시리즈로 묶으면:
- 이전/다음 글 자동 링크
- 시리즈 배너로 전체 구조 노출
- 독자 체류시간 증가 → SEO 긍정 신호

### 3-3. 카테고리 간 연결
예시: 주식 자동매매 글에서 C++ 성능 최적화 글로 연결
> "자동매매 시스템의 주문 처리 속도를 높이려면 [C++ 메모리 풀 구현](/categories/cpp/memory-pool-part1)을 참고하세요."

## 4. 검색엔진 등록 체크리스트

### 4-1. Google Search Console
- URL: https://search.google.com/search-console
- 이미 등록되어 있을 가능성 높음 (Analytics 연동)

**확인할 것:**
- [ ] 사이트맵 제출: `https://tgparkk.github.io/sitemap.xml`
- [ ] 색인 현황 확인: "페이지" → "색인 생성됨" 숫자 확인
- [ ] 색인 안 된 페이지가 있으면 "URL 검사" → "색인 생성 요청"
- [ ] "실적" 탭에서 어떤 검색어로 노출되는지 확인 (콘텐츠 방향 참고)

### 4-2. 네이버 서치어드바이저
- URL: https://searchadvisor.naver.com
- 이미 네이버 인증 파일(naver7f9202010f18738156ff36c2fb418e4c.html)이 있으므로 등록 완료 추정

**확인/추가할 것:**
- [ ] 사이트맵 제출: `https://tgparkk.github.io/sitemap.xml`
- [ ] RSS 피드 제출: `https://tgparkk.github.io/feed.xml`
- [ ] "요청" → "웹 페이지 수집" 에서 주요 글 수동 색인 요청
- [ ] "콘텐츠" → "사이트 최적화" 점수 확인 후 권장 사항 적용

### 4-3. Bing 웹마스터 도구
- URL: https://www.bing.com/webmasters
- 이미 Bing에서 유입이 있으므로 자동 색인 중
- 사이트맵 제출하면 색인 속도 향상

**등록 방법:**
- [ ] Google Search Console 계정으로 간편 가져오기 가능
- [ ] 사이트맵 제출: `https://tgparkk.github.io/sitemap.xml`

### 4-4. 새 글 발행 후 루틴

글을 새로 발행할 때마다:
1. GitHub에 push (자동 배포)
2. Google Search Console → URL 검사 → 색인 요청
3. 네이버 서치어드바이저 → 웹 페이지 수집 요청
4. (선택) 관련 커뮤니티에 공유 (개발자 커뮤니티, 주식 관련 카페 등)
