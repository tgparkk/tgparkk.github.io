## 송신처의 IP주소와 MAC 주소

IP주소는 송신처와 수신처를 지정하기 위해서 필요하다

데이터 통신을 위해서는 수신처랑 송신처를 지정하기 위해 주소가 필요한데, 3계층에서는 이 주소로 IP 주소를 사용한다.
IP 주소는 IP 헤더에 '송신처 IP 주소' 하고 '수신처 IP 주소' 로 기술되는 것

(2계층 이더넷에서 사용되는 MAC 주소도 이더넷 헤더에 '송신처 MAC주소' 와 '수신처 MAC주소')


따라서,
이더넷을 사용해서 IP 데이터그램을 송수신하기 위해서는 ` 4개의 주소` 가 필요하다.

수신처 MAC, 송신처 MAC, 수신처IP, 송신처IP 주소

` 순서대로 진행해보면`
먼저, 송신처 MAC 주소, 이것은 송신처 자신의 MAC 주소인데 이것은 송신할 인터페이스의 MAC 주소를 사용한다.
(MAC 주소는 인터페이스 고유의 주소니까, 송신할 인터페이스 고유의 MAC 주소를 송신처로 사용)

다음으로 IP 주소는 '정적', '동적' 2가지의 종류로 결정된다.

정적인 것은 ` 수동으로 IP주소를 설정` 하는 방법, 네트워크 관리자가 정한 IP주소를 자신의 컴퓨터에 입력하는 것

동적인 것은 IP주소가 자동으로 컴퓨터에 설정되는 것 -> DHCP(Dynamic Host Configuration Protocol) 라고 하는 프로토콜을 사용한다.


## DHCP
` DHCP는 할당할 IP주소를 관리하고, 실제로 할당 작업을 수행하는 서버(Server)하고 할당 받는 클라이언트(Client)로 이루어진다.`

클라이언트가 IP 주소를 '요청'하면 서버가 그에 응해 IP 주소를 '할당' 하는 것이다.

서버는 ` 설정된 IP 주소 풀 중에서 요청한 클라이언트에게 각각 유일한 것이 되도록 주소를 할당하는 거야`
