### 1. "The Anatomy of Trading Breakouts"
**URL**: https://www.investopedia.com/articles/trading/08/trading-breakouts.asp  
**저자**: James Chen (Updated 2025-10-14)  
**로컬 파일**: `research-materials/saved-articles/The_Anatomy_of_Trading_Breakouts.txt`

**핵심 내용**:
- [x] 돌파 전략의 기본 개념
- [x] 진입 조건
- [x] 청산 조건
- [x] 실전 팁

**요약**:
- 돌파 정의: 지지·저항선을 종가 기준으로 돌파하고 거래량이 동반될 때가 신뢰도 ↑.
- 진입: 다중 테스트된 레벨을 기다렸다가 종가가 바깥에서 마감할 때 진입, EOD 확인 권장.
- 목표가: 패턴 높이(채널 폭, 최근 스윙 범위)를 투영하거나 평균 가격 변동 폭을 사용.
- 손절/재진입: 이전 저항=새 지지(또는 반대) 원칙을 활용해 손절선을 명확히 두고, 재테스트 허용.
- 단계 체크리스트(후보 선정→돌파 확인→목표 설정→재테스트 대응→실패 판단→청산)로 감정 통제.

**실전 메모**:
- 거래량이 평균 대비 뚜렷하게 증가할 때만 “진짜” 돌파로 간주.
- 장 마감 전에만 돌파 여부 확정 → 장중 노이즈로 인한 페이크아웃 회피.

## 📚 Axi 자료

### "What is a Breakout Trading Strategy & How to Trade It?"
**URL**: https://www.axi.com/int/blog/education/breakout-trading-strategy  
**저자**: Milan Cutkovic  
**로컬 파일**: `research-materials/saved-articles/What_is_a_Breakout_Trading_Strategy__How_to_Trade_It.txt`

**핵심 내용**:
- [x] 거래량/확인법
- [x] 실전 팁
- [x] 주의사항

**요약**:
- 매수·매도 정지 주문을 저항/지지 돌파 직후에 배치하고, 손절은 직전 레벨 바로 밖에 둬 손실 최소화.
- 지지·저항을 다중 타임프레임으로 그려 돌파 후보를 선별, MT4 차트로 반복 연습 권장.
- 보조 지표: Ichimoku 구름 돌파, RSI 모멘텀/다이버전스로 페이크아웃 필터링.
- 돌파 후 리테스트까지 기다리는 전술과 즉시 진입 전술의 장단점을 비교, 리스크-리워드는 최소 1:2 유지.
- 단점: 빠른 의사결정 요구 + 다수의 페이크아웃 → 엄격한 리스크 관리가 필수.

**실전 메모**:
- RSI가 과매수(또는 과매도)에서 다이버전스가 나오면 돌파 무효 신호로 간주.
- 리테스트 진입을 노릴 경우, 모멘텀 소진 위험이 있으므로 포지션 규모를 절반으로 나누는 분할 진입 전략 고려.

## 📄 학술 자료 요약

### "FinLLM-B: When Large Language Models Meet Financial Breakout Trading"
**arXiv**: 2402.07536v2 (NAACL 2025 산업 트랙)  
**로컬 파일**: `research-materials/saved-papers/FinLLM-B_When_LLM_Meet_Financial_Breakout_Trading.pdf`

**핵심 내용**:
- S&P 500 선물 발자국(footprint) 데이터를 10샷 규모로 정제, S1(방향)·S2(저항)·S3(매수/매도 힘) 세부 과제로 분해.
- GPT-3.5를 기반으로 한 FinLLM-B가 다단계 구조에서 평균 정확도 84.8%, 퍼펙션 비율 59.45%로 GPT-4 대비 +42.38%p.
- S2 정확도는 89.4%로, 저항 레벨 수치 예측에서 표준편차 0.06까지 억제.
- 데이터 수가 2샷→10샷으로 늘어나면 정확도가 57.5%→84.8%로 포화, 데이터 부족 환경에 LLM 접근이 유효함을 시사.
- 실무 적용 시 합리적 보고서를 요구하는 금융 도메인 특성에 맞춰 근거 사슬을 강제하는 멀티 스테이지 프롬프트 설계가 핵심.

**적용 아이디어**:
- 발자국 차트 + 주문 흐름 데이터를 한국 시장 호가 데이터로 치환하여 로컬 모델 검증 가능.
- 기존 백테스트 파이프라인에 “S1/S2/S3 세부 태스크 → 보고서 생성” 구조를 접목해 설명 가능한 시그널 필터로 활용.
